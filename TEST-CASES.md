# تست‌کیس‌های داشبورد (نسخه‌ی چت‌محور + اجرای خودکار + گزارش)

> هر تست‌کیس: **پیش‌نیاز / مرحله‌ها / نتیجه‌ی موردانتظار**.
> اگر موردی مطابق نبود، شماره‌ی تست‌کیس را بگو تا درستش کنم.

## پیش‌نیازها
- داشبورد در حال اجرا: `node dashboard/server.js` → http://localhost:8090
- برای چت/ایجنت/گزارشِ واقعی: پل مدل هم روشن باشد → `node claude-bridge.js` (پورت 8787)
- برای تسک/برد/مرج واقعی: توکن‌های Trello و GitHub در تنظیمات وارد شده باشند
- توجه: بدون پل، چت پیام «اتصال به پل برقرار نشد» می‌دهد (طبیعی است؛ بقیه‌ی UI کار می‌کند)

## ۱) دکمه پلی/استاپ + دیالوگ + اجرای واقعی

**TC1.1 — تغییر حالت با دیالوگ**
- بالای برد دکمه‌ی «شروع اجرای خودکار» (آیکن ▶) را بزن.
- ✅ یک **دیالوگ استایل‌دار** (نه دیالوگ مرورگر) باز شود با عنوان «شروع اجرای خودکار» و متن «از این پس تسک‌ها خودکار اجرا، ریویو و به develop مرج می‌شوند…».
- تأیید کن → دکمه قرمز و پالس‌دار شود، برچسبش «توقف اجرای خودکار»، آیکن ⏹، و راهنمای زیر آن به حالت روشن تغییر کند.

**TC1.2 — قفل اجرای دستی در حالت خودکار**
- وقتی خودکار روشن است، یک کارت در «در صف» را باز کن.
- ✅ دکمه‌ی اجرای دستی کارت قفل (🔒) باشد.

**TC1.3 — ماندگاری وضعیت**
- صفحه را رفرش کن.
- ✅ دکمه همچنان «توقف اجرای خودکار» (روشن) بماند (روی دیسک ذخیره شده).

**TC1.4 — اجرای واقعی تا مرج (نیازمند Trello+GitHub+پل)**
- چند تسک روی برد باشد، خودکار را روشن کن، ~۲۰ ثانیه صبر کن.
- ✅ کارت «در صف» خودش اجرا شود → PR ساخته شود → در تیک بعدی ریویو شود → در صورت سبزبودن CI به develop مرج و کارت به «تمام‌شده» برود؛ همه بدون کلیک دستی.

## ۲) گزارش وضعیت (PO) روی STOP و خوانده‌شدن روی START

**TC2.1 — گزارش هنگام توقف**
- خودکار را روشن و سپس **خاموش** کن (توقف).
- ✅ بعد از تأیید، یک حباب لودینگ بیاید و سپس پیام گزارش با انیمیشن تایپ نمایش داده شود: «📋 گزارش وضعیت پروژه (پیشرفت تقریبی ٪… )…» یا اگر برد خالی است «هیچ تسکی روی برد این پروژه نیست».
- ✅ گزارش در چت ذخیره شود (بعد از رفرش هم بماند).

**TC2.2 — ذخیره روی دیسک**
- بعد از TC2.1، فایل `data/projects/<projectId>/report.json` ساخته/به‌روز شده باشد.

**TC2.3 — خواندن گزارش توسط دولوپر/تک‌لید روی START (نیازمند پل+Trello)**
- بعد از داشتن گزارش، خودکار را دوباره روشن کن (یا یک تسک را دستی اجرا کن).
- ✅ در اجرای دولوپر و ریویوی تک‌لید، متن گزارش به‌عنوان «گزارش وضعیت قبلی پروژه» به بریف داده شود تا کار منسجم ادامه پیدا کند (در عمل: ادامه‌ی کار با توجه به وضعیت قبلی، نه از صفر).

## ۳) دیالوگ‌ها همه‌جا استایل تنظیمات (نه دیفالت مرورگر)

**TC3.1 — پلی/استاپ** → دیالوگ استایل‌دار (TC1.1). ✅
**TC3.2 — حذف پروژه** → آیکن سطل/منوی پروژه → حذف → دیالوگ استایل‌دار با دکمه‌ی قرمز «حذف» و «انصراف». ✅
**TC3.3 — حذف چت** → کنار سشن → حذف → همان دیالوگ قرمز. ✅
**TC3.4 — اجرای تسک کارت** → روی کارت دکمه‌ی اجرا → دیالوگ تأیید استایل‌دار. ✅
**TC3.5 — پوشه‌ی حذف‌شده** → پوشه‌ی یک پروژه‌ی لوکال را دستی پاک کن → تب ریپو → دیالوگ استایل‌دار «حذف شده؟». ✅
**TC3.6 — خطای checkout** → checkout ناموفق → پیام خطا به‌صورت دیالوگ استایل‌دار (تک‌دکمه «تأیید»). ✅
- در همه‌ی موارد: **هیچ دیالوگ پیش‌فرض مرورگر (alert/confirm)** نباید دیده شود.
- Esc = انصراف، Enter = تأیید، کلیک روی بک‌دراپ = انصراف.

## ۴) سه حالت کامپوزر: محصول / چت / Agent

**TC4.1 — وجود سه حالت**
- بالای کادر ارسال سه تب باشد: **محصول** (پیش‌فرض، فعال)، **چت**، **Agent**.

**TC4.2 — حالت محصول (Product Owner)**
- روی «محصول» متن بنویس (مثل «یک صفحه ورود بساز») و بفرست.
- ✅ پیام به PO می‌رود؛ یا جواب می‌دهد یا تسک می‌سازد و به برد اضافه می‌کند (نیازمند پل/Trello). placeholder: «چی بسازیم؟ بنویس…».

**TC4.3 — حالت چت (پرسش/پاسخ، بدون تغییر فایل)**
- تب «چت» را بزن → placeholder «یک سؤال بپرس…»، دکمه‌ی پیوست تصویر ظاهر شود.
- یک سؤال بپرس → ✅ پاسخ متنی بیاید و **هیچ فایلی/تسکی تغییر نکند** (read-only).

**TC4.4 — حالت Agent (کار روی کد)**
- تب «Agent» را بزن → placeholder «به agent بگو…»، چک‌باکس «پوش بعد از کد» و پیوست تصویر دیده شود.
- یک تسک کدنویسی بده → ✅ ایجنت روی برنچ فعلی کد بزند و (در صورت تیک پوش) کامیت/پوش کند (نیازمند پل + ریپوی کلون‌شده).

**TC4.5 — تعویض حالت**
- بین تب‌ها جابه‌جا شو → placeholder، خط راهنما و ردیف ابزار هر حالت درست عوض شود.

## ۵) تسک‌ها از فایل (منبع اصلی) + سینک با ترلو روی سوییچ

**TC5.1 — سینک هنگام سوییچ (نیازمند Trello)**
- در پروژه‌ی A چند تسک بساز (حالت محصول)، به پروژه‌ی B و دوباره به A سوییچ کن.
- ✅ تسک‌های A دوباره روی برد دیده شوند (از فایل `tasks.json` خوانده و کارت‌های گمشده روی ترلو دوباره ساخته شوند).

**TC5.2 — ماندگاری**
- فایل `data/projects/<A>/tasks.json` شامل تسک‌های A با `cardId` باشد.

## ۸) انیمیشن پاسخ مثل GPT + دکمه کپی

**TC8.1 — انیمیشن تایپ**
- یک پاسخ بگیر → ✅ متن **حرف‌به‌حرف با کرسر چشمک‌زن** تایپ شود (نه یک‌باره) و وسط انیمیشن با رفرش پس‌زمینه قطع/پرش نکند.

**TC8.2 — دکمه کپی**
- روی هر پیام مدل/سیستم نشانگر را ببر → دکمه‌ی «کپی» با آیکن ظاهر شود.
- بزن → ✅ متن کپی شود و برچسب لحظه‌ای «کپی شد» شود.
- ✅ پیام‌های کاربر دکمه‌ی کپی نداشته باشند.
